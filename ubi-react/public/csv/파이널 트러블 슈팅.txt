❌ 로그인 여부에 따라 지자체 혜택이 보이지 않음
문제 내용:
로그인된 상태에서는 메인 페이지에서 종로구 혜택이 보이지만, FacilityDetailPage에서는 동일한 지역이라도 📭 종로구의 복지 혜택 정보가 없습니다. 메시지만 출력됨.

원인 분석:

로그인된 사용자에 한해 selectedCity, selectedDistrict가 store에서 전달됨.

하지만 FacilityDetailPage에서 welfare 데이터를 조회하는 API 호출이 로그인 세션이나 토큰을 기반으로 하지 않고, 단순 지역명 기반이어서 필터링 기준 불일치 발생.

또한, API 호출 후 데이터를 지역별로 분류할 때 jurMnofNm(주관 시군구명)으로만 분기했는데, 이 필드에 "종로구"와 완전히 일치하지 않는 경우가 있어 조건 불일치로 간주됨.

해결 방법:

로그인 여부와 관계없이 selectedCity/selectedDistrict 값을 받아서 jurMnofNm 또는 지역명 기준으로 포함(includes) 방식 필터 적용.

최종 적용 코드 예시:

js
코드 복사
const matched = groupedWelfareList[selectedDistrict] 
             || groupedWelfareList.find((key) => key.includes(selectedDistrict));
추가 조치:

비로그인 사용자도 지역 선택 시 서비스 정상 출력되도록 개선.

향후 로그인 여부와 무관하게 지역 선택값을 통일해서 처리 예정.

2. ❌ 복지 혜택이 "정보 없음"으로만 출력됨
문제 내용:
FacilityDetailPage.jsx에서 혜택 정보는 항상 "정보가 없습니다"로 출력됨.

원인 분석:

메인 페이지에서는 welfare 데이터를 API로 받아오고 있었으나, 상세 페이지에서는 welfare API를 호출하지 않음.

또한 혜택 컴포넌트 WelfareBenefitView가 실제 데이터를 props로 받지 않고 있었음.

해결 방법:

FacilityDetailPage.jsx 내에서 /api/welfare-curl/welfare-list/all 호출하여 welfare 데이터를 받아옴.

지역별로 jurMnofNm을 기준으로 그룹화 후 selectedDistrict와 일치 여부를 기준으로 필터링하여 전달.

3. ❌ import 경로 오류 발생
문제 내용:
다음과 같은 오류 발생:
Failed to resolve import "./FacilityCard"
Failed to resolve import "../../components/welfare/WelfareBenefitView"

원인 분석:

경로가 잘못 지정되어 실제 파일 위치와 불일치하거나 파일이 존재하지 않음.

해결 방법:

FacilityCard → ../../components/welfarefacility/FacilityCard

WelfareBenefitView → ../../components/welfare/WelfareBenefitView
정확한 상대경로로 수정하여 해결.

요구 내용 요약:

복지유형: 라디오 버튼 (예: 지자체 혜택, 지자체 시설)

UI 전체 재구성.

검색어, 카테고리, 복지유형을 모두 적용한 종합 필터링 방식 구현.
